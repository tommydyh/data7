# 🔧 Streamlit Cloud 部署失败 - 修复指南

## ❓ 问题原因

从截图看，GitHub Actions CI 测试失败，主要原因：

1. **硬编码的 Windows 路径** - `D:\cursor data 2\...` 在云端无法工作
2. **缺少依赖** - `joblib`, `threadpoolctl` 等缺失
3. **secrets.toml 问题** - 文件未正确上传

## ⚠️ 数据文件问题

由于 CSV 数据文件较大，可能需要特殊处理：

### 方案 A：上传小数据样本（推荐测试用）

1. 压缩或减少数据文件
2. 上传到 GitHub
3. 或使用公开数据源

### 方案 B：使用云存储

将数据文件上传到：
- AWS S3
- Google Cloud Storage
- Azure Blob Storage

然后在代码中修改路径。

## 🔍 如果仍然失败

### 查看详细错误日志

在 Streamlit Cloud 中：
1. 进入您的应用
2. 点击 **"Logs"** 或 **"Deploy logs"**
3. 查看具体错误信息

### 常见错误

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| `ModuleNotFoundError` | 缺少依赖 | 更新 requirements.txt |
| `FileNotFoundError` | 数据文件不存在 | 检查数据文件路径 |
| `PermissionError` | 权限问题 | 检查文件权限 |
